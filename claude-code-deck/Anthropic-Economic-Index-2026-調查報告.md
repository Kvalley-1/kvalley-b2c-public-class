# 一、March 2026 Report：Learning Curves（最新最重要）

## 1.1 基本資訊

| 項目 | 內容 |
|---|---|
| 完整標題 | Anthropic Economic Index report: Learning curves |
| 副標 | 追蹤 Claude 在經濟中的使用模式 |
| 發布日 | 2026-03-24 |
| 樣本 | Claude.ai + 1P API 各 100 萬對話 |
| 時間範圍 | 2026-02-05~12（8 天） |
| 隱私方法 | CLIO 隱私保護分析（聚合層級，不洩露個別內容） |
| 職業分類框架 | O*NET（2019 版）— 美國勞工部 900+ 職業 20,000 work tasks |

## 1.2 三大核心發現

1. **增強型互動輕微上升**（AI 補充用戶能力 vs 指令型 vs 自動化）
2. **Claude.ai 使用多樣化**：前 10 大任務佔比從 24% 降至 19%
3. **長期用戶更成功**：6 個月+用戶對話成功率高 10%

## 1.3 Claude.ai 使用類別分佈（按 O*NET 大類）

| O*NET 大類 | 使用佔比 | 主要任務 |
|---|---|---|
| **計算機與數學** | **35%** | 軟體開發、程式設計、code review |
| **管理職業** | **5%**（從 11 月 3% 上升） | 投資備忘錄、客戶回應 |
| **教育與圖書館** | **15%** | 課業協助、課程開發 |
| **藝術、設計、娛樂、體育、媒體** | **11%** | 文案編輯、虛構創作 |
| **辦公與行政支援** | **8%**（API 端 15%） | 行政流程 |
| **個人使用** | **42%**（11 月 35%）| 體育查詢、產品比較、家務維護 |
| **課程使用** | **12%**（11 月 19%） | 學生作業 |

## 1.4 11 月 → 2 月 趨勢變化（5 個月變動）

| 課程使用佔比 | 19% | 12% | ↓7pp |

| 平均教育年數 | 12.2 年 | 11.9 年 | ↓0.3 年 |
| 人類完成時間 | — | — | ↓2 分鐘 |

## 1.5 新興自動化工作流（2 月 vs 3 個月前**至少翻倍**）

### 商業銷售與拓展自動化
- 銷售賦能內容生成
- B2B 潛在客戶資格審查研究
- 客戶資料豐富化
- 冷郵件起草

### 自動化交易與市場運營
- 監控市場或頭寸
- 提議具體投資
- 向交易員通知市場狀況

## 1.6 API vs Claude.ai 對比

| 特徵 | Claude.ai | 1P API |
|---|---|---|
| Coding 工作佔比 | 35%（下降 18%） | 上升（API 中計算機+14pp） |
| 任務集中度 | 下降（前 10 從 24%→19%） | 平穩（前 10 仍 33%） |
| 互動類型 | 增強型略升 | 指令型主導 |
| Opus 模型選擇敏感度 | 1.5pp/$10 hr 薪資 | 2.8pp/$10 hr 薪資（敏感度 2x） |

## 1.7 學習曲線：高資歷 vs 新用戶（6 個月+用戶 vs 新用戶）

| 特徵 | 高資歷 | 新用戶 | 差異 |
|---|---|---|---|
| 工作用途佔比 | 較高 | 較低 | +7pp |
| 個人使用佔比 | 38% | 44% | -6pp |
| 任務集中度（前 10） | 20.7% | 22.2% | 更分散 |
| 對話成功率 | 高 10% | 基準 | +10pp（控制後仍 +4pp） |
| 教育年數 | 每年 +1 | 基準 | 線性 |

**解釋**：高資歷用戶習得更好的「AI 提示技巧」，**邊做邊學**假說獲支持。

## 1.8 模型選擇：Opus 使用率（高薪職業偏好高推理模型）

| 職業 | Opus 使用率 | 薪資水平 |
|---|---|---|
| 計算機與數學 | **55%**（+4pp vs 平均） | 高薪 |
| 軟體開發者 | 34% | 高薪 |
| 教育職業 | 45%（-7pp） | 中薪 |
| 輔導職業 | 12% | 低薪 |

**規律**：每 +$10 hr 薪資 → Opus 使用率 +1.5pp（Claude.ai）/ +2.8pp（API）

**相關性**：人均 GDP 提高 1% → AUI 增加 0.7%

## 2.6 美國州級收斂

- Gini 係數：0.37（8 月）→ 0.32（11 月）
- 預測 2-5 年達各州平衡
- **比 20 世紀技術擴散快 10 倍**
- 驅動：州內電腦與數學職業工作者每+1% → 使用率 +0.36%

## 2.7 生產力提升（Speedup）與教育程度

### Claude.ai
- 高中 12 年教育任務：**9 倍 speedup**
- 大學 16 年教育任務：**12 倍 speedup**

### 成功率與複雜度權衡
- 低複雜度任務：70% 成功
- 大學程度任務：66% 成功

### 任務範圍（Task Horizon）— 50% 成功率對應任務時長
- API：**3 小時**任務
- Claude.ai：**19 小時**任務（多輪對話設計成功率下降較緩）

## 2.8 經濟生產力預測

| 情境 | Claude.ai | API |
|---|---|---|
| 未調整成功率 | **1.8pp 年增長** | 1.8pp |
| 調整成功率 | 1.2pp | 1.0pp |
| 任務互補（σ=0.5） | 0.8pp | — |
| 任務可替代（σ=1.5） | 2.2-2.6pp | — |

## 2.9 工作去技能化分析

**整體發現**：移除 Claude 能做的任務後，剩餘工作平均教育要求降低
**原因**：Claude 傾向覆蓋高教育程度任務（14.4 年 vs 全經濟平均 13.2 年）

### 去技能化範例
- **技術文案撰稿人**：失去「分析特定領域發展確定修訂必要性」（18.7 年）
- **旅遊代理商**：失去「規劃、描述、安排與銷售行程套餐」（13.5 年）

### 升技能化範例
- **房地產經理**：保留複雜談判和利益相關人管理，失去行政記錄維護

## 2.10 高覆蓋職業範例

- 數據輸入員：最耗時任務（讀取並輸入源文檔）成功率高
- 醫學轉錄員、放射科醫師：最耗時工作高成功率
- 微生物學家：覆蓋率低（無法執行實驗室操作）

## 2.11 政策含義（原文引用）

> 「為使 AI 在全球惠及使用者，僅擴大使用存取不足——開發使用者有效運用 AI 的人力資本，特別是在低收入經濟體，至為關鍵。」
