## M001 Reddit 工程師一夜燒掉 $6,000（個人燒錢趣聞）

- **類型**：個人警示 / 趣聞
- **故事**：開發者設 Claude Code 每 30 分鐘自動檢查更新，整夜運行；每次都重新上傳完整 context window → 一天跑了 **48 次**；Anthropic 帳單延遲數天才更新 → 早上收到帳單時**已經 $6,000**
- **重點**：Anthropic 用量儀表板的延遲，讓「跑爆預算」事後才知道
- **比擬**：美國軟體工程師中位數月薪的 1.5 倍——一個晚上沒了
- **教學用途**：S1 個人尺度燒錢警示（不是公司，是「你個人」也能燒）
- **來源**：https://www.makeuseof.com/someone-left-claude-code-running-overnight-and-it-cost-6000/

## M002 視覺提案：員工 vs Agent 比例尺（橫條圖）

- **類型**：視覺設計提案
- **格式**：橫條圖單頁
- **內容範例**：
```
McKinsey       人 ████████████████ 40,000   Agent ██████████ 25,000   (1.6:1)
Customers Bank 人 ▌870              Agent ▍500              (1.7:1)
EY (tax)       人 ████████ 80,000   Agent ▏150              (533:1) → 2028: 100,000
Moderna        人 ▎5,800            Agent ▌3,000+           (1.9:1)
Salesforce(支援) 人 ▍5,000          Agent (4,000 砍掉)      ← 直接負成長
BoA            人 ████████ 213,000  Agent (Erica 5,800萬/月互動)
Foxconn(目標)  人 ████████ 826,000  Agent (3 段式 80% AI 流程化)
```
- **設計重點**：兩個對比色（藍 = 人、橘紅 = agent），每條後附 ρ 員工比 + 目標年份。**所有公司放同一張**，讓老闆一眼看到自己公司「沒在這張圖上」的尷尬。
- **教學用途**：S1 翻轉敘事核心視覺

## M003 視覺提案：時間軸「2 年內翻轉」（橫向 timeline）

- **類型**：視覺設計提案
- **格式**：橫向 4 欄 timeline

| | 2023 末 | 2024 末 | 2025 末 | 2026 中 |
|---|---|---|---|---|
| McKinsey agent 數 | 0 | ~5,000 | ~15,000 | **25,000** |
| Moderna GPT 數 | 0 | 750 | 3,000 | 3,000+ |
| EY agents | 0 | 50 | 150 | 1,000 開發中 |
| Google AI 寫程式 % | <10% | 25% | 50% | **75%** |
| BoA Erica 累積互動 | 1.5B | 2.5B | 3.2B | 接近 4B |

- **設計重點**：每欄寬度一樣，但**柱狀體高度幾何級數成長**，讓人視覺感受「過去兩年 = 翻轉發生的兩年」
- **教學用途**：S1 翻轉敘事的時間驅動證據

## M004 視覺提案：亞洲成熟度熱力地圖

- **類型**：視覺設計提案
- **格式**：亞洲地圖 + 顏色等級
```
🟥 深紅（領先）：南韓 41% 跨部門規模化（4 大集團同步壓注）
🟧 橘（積極）：中國（Qwen3.7-Max 35hr 自主）、新加坡（國家級 testbed）
🟨 黃（中等）：台灣（TSMC/Foxconn 製造業領先，服務業跟進中）
🟩 綠（觀望）：泰國、馬來西亞、印度（高 GDP 成長帶動但散）
⬜ 灰（落後）：日本 41% 沒計畫
```
- **設計重點**：地圖 + 每國上有 1 個代表案例 + ρ 員工比；殺手收尾語：「**台灣現在比日本快，但比南韓慢。三年後在哪邊？取決於今天**」
- **教學用途**：S1 亞洲對照段落視覺
